# ops_tool 项目说明

## 作用范围

本文件适用于 `D:\my-tools\ops_tool` 目录下的所有内容。

## 项目用途

`ops_tool` 是一个本地单机使用的运营工具。

当前包含：

- `scripts/`：Python 核心处理逻辑和脚本版入口
- `web/`：本地网页界面和本地任务服务
- `data/`：任务输入文件和中间文件
- `output/`：生成结果
- `docs/`：需求文档和实现计划文档

这个工具的主要目标是：

- 处理 Word 文章
- 解析 sitemap
- 生成 prompt
- 把原本需要手动跑脚本的流程变得更简单

## 当前产品方向

- 优先做轻量、本地可用的工具，不做重平台
- 保留脚本版可继续使用
- 前端版保持简单、实用
- 默认采用顺序执行任务，不做并行，除非用户明确要求
- 每个任务都保留自己的日志、prompt 和输出路径

## 关键用户预期

- 本地文件选择要简单直接
- prompt 复制要明显且稳定
- 一个任务运行时，后续任务仍然可以继续加入等待队列
- 错误提示要让不懂代码的人也能看懂

## 开发时的重要约定

- 优先复用 Python 里的业务处理逻辑，不要在 Node 或浏览器里重复实现一套
- 浏览器层重点负责：
  - 提交任务
  - 展示队列
  - 展示日志
  - 展示和复制 prompt
- 队列状态先保持简单，除非用户明确要求持久化或更复杂的调度
- 在工具仍处于迭代阶段时，避免做大规模无关重构

## 运行与验证

### Python 测试

```powershell
pytest D:\my-tools\ops_tool\scripts\test_pipeline.py -v
```

### 网页队列测试

```powershell
cd D:\my-tools\ops_tool\web
npm test
```

### 启动前端版

```powershell
cd D:\my-tools\ops_tool\web
npm run dev
```

打开地址：

```text
http://127.0.0.1:3210
```
